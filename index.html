05Proceso

Cuatro pasos. Cero improvisación.

[ 01 ] Cazar el contexto Inmersión en el negocio del cliente. Brief, objetivos, audiencia y competencia. Antes de diseñar una sola pantalla, entiendo qué necesita resolver la web. Ver fase →
Brief del cliente · Análisis de competencia directa e indirecta · Definición de audiencia · Moodboard de referentes · Duración estimada: 3–5 días hábiles · Entregable: documento de diagnóstico.
[ 02 ] Marcar el rastro Arquitectura de información, user journey y wireframes. La estructura antes de los colores. El mapa antes del salto. Ver fase →
User journey · Arquitectura de información · Wireframes de baja y media fidelidad · Estructura de contenidos · Duración: 4–7 días hábiles · Entregable: wireframes aprobados.
[ 03 ] Diseñar con instinto Sistema visual, paleta, tipografía y diseño de pantallas. Desktop y mobile. Prototipo navegable para validar antes de escribir una línea de código. Ver fase →
Sistema visual completo · Diseño UI desktop + mobile · Prototipo en Figma · Hasta 2 rondas de revisión incluidas · Duración: 7–14 días hábiles · Entregable: diseño final aprobado.
[ 04 ] Defender la presa Construcción, optimización y deploy. Performance, SEO básico, accesibilidad y QA cross-device. La web queda funcionando sola, con argumentos. Ver fase →
Desarrollo del sitio · Optimización de performance · SEO básico (meta tags, estructura semántica, Open Graph) · QA cross-browser y cross-device · Deploy en Vercel o hosting del cliente · Soporte de lanzamiento: 7 días post-entrega.
